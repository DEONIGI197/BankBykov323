### Разработка модульных тестов

В рамках работы было создано **5 тестовых методов**, покрывающих функциональность методов `Debit` и `Credit` класса `BankAccount`. Тестирование проводилось методом «белого ящика», что позволило проанализировать внутреннюю логику программы.

**Сценарии тестирования:**

- Проверка корректности списания средств.
- Проверка корректности пополнения счета.
- Обработка исключений при попытке списания отрицательной суммы.
- Обработка исключений при попытке списания суммы, превышающей баланс.
- Обработка исключений при попытке пополнения счета на отрицательную сумму.

### Результат выполнения тестов

Все разработанные тесты были успешно пройдены. Это подтверждает корректность реализации методов и надежность обработки как стандартных, так и исключительных ситуаций.

### Анализ результатов

В ходе выполнения практической работы были решены следующие задачи:

1. Создана структура из двух проектов: основной `Bank` и проект с модульными тестами `BankTests`.
2. Реализовано модульное тестирование, покрывающее ключевые сценарии работы с банковским счетом.
3. Благодаря применению метода «белого ящика» в коде метода `Debit` была обнаружена логическая ошибка, связанная с неверным знаком операции (использование `+=` вместо `-=`).
4. После обнаружения ошибки код был исправлен. Также был проведен рефакторинг тестов: добавлены константы для ожидаемых значений и улучшены проверки сообщений исключений.
5. Итоговая проверка подтвердила полную работоспособность и корректность программы.

**Вывод**

Автоматизированное тестирование является неотъемлемой частью разработки качественного программного обеспечения. Данная практическая работа наглядно демонстрирует, как своевременное написание unit-тестов помогает не только подтвердить работоспособность кода, но и выявить скрытые дефекты на ранних этапах, что значительно повышает надежность и устойчивость приложения.
